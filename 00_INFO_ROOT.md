## Document Descriptions

### README.md
**What**: User-facing introduction and setup guide  
**Contains**: Installation steps, quick start, CLI commands, basic configuration  
**When to read**: First contact with the project, getting started

### CAPABILITIES.md
**What**: Comprehensive product/technical capabilities reference  
**Contains**: Feature catalog, API endpoints, configuration schemas, integration points, deployment guide  
**When to read**: Evaluating features, planning integrations, handover documentation

### KNOWLEDGE.md
**What**: Developer knowledge base and technical reference  
**Contains**: Architecture decisions, data flows, code patterns, gotchas, internal APIs  
**When to read**: Before contributing code, debugging, understanding internals

### PLAN.md
**What**: Implementation plan and progress tracking  
**Contains**: Completed features, in-progress work, future roadmap, task status  
**When to read**: Understanding project state, finding what's done vs pending

## Maintenance

When updating documentation:
1. **User-facing changes** → README.md
2. **New features/APIs** → CAPABILITIES.md
3. **Implementation details** → KNOWLEDGE.md
4. **Task completion** → PLAN.md

Update the "Last Updated" timestamp in each file when making changes.
